Understanding how Selecty impacts your store's SEO is important for maintaining search visibility while providing an excellent international customer experience.

## Selecty's Role in SEO

<Info>
**Important:** Selecty is not an SEO optimization tool. It's a user experience enhancement tool for international stores.
</Info>

### What Shopify Handles

Shopify automatically manages all critical SEO elements for multilingual and multi-market stores:

- **Sitemap generation** - XML sitemaps for all markets and languages
- **Hreflang tags** - Language and region-specific content signals
- **Canonical URLs** - Prevent duplicate content issues
- **Structured data** - Rich snippets and schema markup
- **Meta tags** - Titles, descriptions, and language declarations

### What Selecty Does

Selecty provides user interface elements to help customers navigate between markets:

- Language, currency, and market switchers
- Recommendation banners for locale suggestions
- Optional automatic redirects
- Multiple store connections

**Selecty does not interfere with Shopify's SEO features.** Your theme and Shopify's platform handle all SEO-related technical implementation.

## Impact of Automatic Redirects

The only Selecty feature that can potentially affect SEO is **automatic redirects** (Markets Redirect and cross-store redirects).

### Why Redirects Can Hurt SEO

According to [Google's guidelines on multilingual sites](https://developers.google.com/search/docs/specialty/international/managing-multi-regional-sites):

> "Avoid automatic redirection based on the user's perceived language. These redirections could prevent users (and search engines) from viewing all the versions of your site."

**Problems with automatic redirects:**

1. **Incomplete Indexing** - Search bots may not discover all language versions
2. **Hreflang Issues** - Language alternate tags may not be properly processed
3. **Regional Content** - Different markets' content may not appear in search results
4. **User Signals** - High bounce rates if redirects are incorrect

### Bot Detection Limitations

Selecty includes bot detection to prevent redirecting search engine crawlers. However:

<Warning>
**No bot detection is 100% accurate**

Modern programming languages cannot guarantee perfect bot identification. Some search bots may be redirected, and some human visitors may not be. This inherent limitation affects all bot detection systems.
</Warning>

## Multi-Domain Considerations

If using separate domains for different markets (example.com, example.fr, example.de), pay special attention to SEO configuration.

### Canonical Tags

**What they are:** HTML tags that tell search engines which version of a page is the "main" one

**Who manages them:** Your Shopify theme, not Selecty

**Why they matter:** Prevent duplicate content penalties when markets have similar content

<Warning>
**Theme Responsibility**

Canonical tags are generated by your theme. Selecty does not manage or interfere with canonical URLs. If you have canonical tag issues, contact your theme developer or consult Shopify support.
</Warning>

### Domain-Specific Best Practices

1. **Each domain should target its region** - Use local domain extensions (.fr, .de, .co.uk)
2. **Geo-targeting in Search Console** - Set each domain's target country
3. **Local hosting** - Consider CDN or hosting closer to target region
4. **Local backlinks** - Build links from sites in target region
5. **Localized content** - Unique content for each market, not just translations

## Shopify Markets and SEO

Shopify Markets provides built-in SEO features that work independently of Selecty:

### Automatic SEO Features

<Tabs>
  <Tab title="Hreflang Tags">
    **What it does:** Tells search engines about language/region alternatives

    **How it works:**
    - Automatically added to page head
    - Links all language versions together
    - Helps serve right version to searchers

    **Example:**
    ```html
    <link rel="alternate" hreflang="en-us" href="https://example.com" />
    <link rel="alternate" hreflang="fr-fr" href="https://example.com/fr" />
    <link rel="alternate" hreflang="de-de" href="https://example.com/de" />
    ```
  </Tab>

  <Tab title="Sitemaps">
    **What it does:** Provides search engines with a map of all pages

    **How it works:**
    - Automatically generated at /sitemap.xml
    - Includes all markets and languages
    - Updates when content changes

    **Coverage:**
    - Products
    - Collections
    - Pages
    - Blog posts
    - All market variations
  </Tab>

  <Tab title="Structured Data">
    **What it does:** Provides rich information about products and content

    **How it works:**
    - JSON-LD structured data
    - Product information
    - Pricing and availability
    - Reviews and ratings

    **Benefits:**
    - Rich snippets in search results
    - Better product visibility
    - Enhanced click-through rates
  </Tab>
</Tabs>

## Common SEO Questions

<AccordionGroup>
  <Accordion title="Will Selecty hurt my search rankings?">
    No, Selecty itself doesn't affect rankings. The app enhances user experience without interfering with Shopify's SEO features. However, if you enable automatic redirects, that could impact SEO negatively.
  </Accordion>

  <Accordion title="Should I use auto-redirect or recommendation banners?">
    **Recommendation banners** are better for SEO. They don't prevent search engines from accessing all versions, and Google specifically recommends against automatic redirects for multilingual sites.
  </Accordion>

  <Accordion title="Do I need to do anything for hreflang tags?">
    No. Shopify automatically handles hreflang tags for Markets. Selecty doesn't interfere with this functionality.
  </Accordion>

  <Accordion title="How do canonical tags work with Selecty?">
    Canonical tags are managed by your Shopify theme, not Selecty. The app doesn't add, modify, or interfere with canonical URLs.
  </Accordion>

  <Accordion title="Can I use Selecty with multi-domain setup?">
    Yes. Selecty works with multi-domain Markets. Just ensure your theme properly implements canonical tags for each domain.
  </Accordion>
</AccordionGroup>
